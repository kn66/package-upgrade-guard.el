* Overview

Package Upgrade Guard is an Emacs security tool for reviewing package modifications and detecting supply-chain changes introduced by upgrades. It supports ELPA/MELPA packages and, on Emacs versions with ~package-vc~, git-based VC (version control) package upgrades. Direct installation of a package that is not already installed proceeds without a review prompt.

⚠️ Note: Starting with Emacs 31, similar package-upgrade review functionality is available in upstream Emacs. This package currently declares tested support for Emacs 30.2 or later; older Emacs support is not claimed until it is covered by CI or manual verification.

* Features

- 🔍 Shows diffs for package upgrades before installation
- 📦 Supports both tarball packages (ELPA/MELPA) and VC packages (git repositories)
- 🛡️ Security-focused review process with user approval prompts
- 🔄 Reviews modifications initiated by ~package-install~, ~package-upgrade~, ~package-upgrade-all~, and package menu operations
- 🔗 Reviews new dependencies introduced by package upgrades
- 📊 Displays file-by-file changes, new files, and deleted files
- 🔒 Verifies that installed archive bytes match the reviewed SHA-256 digest
- 📌 Installs the exact reviewed VC commit without a second network fetch
- ⚡ Efficient TAR extraction for package analysis

* Installation

1. Clone this repository, or install it with ~use-package~'s ~:vc~ support.
2. If installing manually, add the repository directory to your load path.  Copying only ~package-upgrade-guard.el~ is not sufficient because the package is split across companion ~package-upgrade-guard-*.el~ files.
3. Add to your Emacs configuration:

#+BEGIN_SRC emacs-lisp
  (use-package package-upgrade-guard
    :vc ( :url "https://github.com/kn66/package-upgrade-guard.el.git" :rev :newest)
    :config
    (package-upgrade-guard-mode +1))
#+END_SRC

* Usage

Once enabled, Package Upgrade Guard automatically intercepts:

- ~M-x package-install~ - Reviews the diff when it modifies an installed package; new installations proceed directly
- ~M-x package-upgrade~ - Shows diff before upgrading individual packages
- ~M-x package-upgrade-all~ - Reviews each package individually
- Package menu operations (upgrades marked with ~U~ are reviewed; new installs marked with ~I~ proceed directly)

** Interactive Workflow

1. When you attempt to upgrade a package, a diff buffer appears
2. Review the changes in the displayed diff
3. Respond to the approval prompt:
   - ~yes~ - Proceed with installation/upgrade
   - ~no~ - Cancel the operation

** VC Packages

For git-based VC packages, the tool shows:
- Git status
- New commits to be pulled
- Detailed diff of changes from the configured upstream branch

** Regular Packages

For ELPA/MELPA packages, the tool shows:
- File-by-file unified diffs
- New files added
- Files removed
- Version information
- New dependency packages introduced by the upgrade

* Configuration

** Customization Variables

#+BEGIN_SRC emacs-lisp
;; Enable/disable the security checker
(setopt package-upgrade-guard-enabled t)

;; Custom base directory for temporary package review sessions
(setopt package-upgrade-guard-temp-dir "/path/to/temp/dir")

;; Show the complete available diff and highlight configured sensitive
;; patterns.  Unmatched changes remain visible; heuristic detection is used
;; only to prioritize review.
(setopt package-upgrade-guard-diff-mode 'security)

;; Reject unexpectedly large downloads and flag oversized reviews as incomplete.
(setopt package-upgrade-guard-max-download-size (* 50 1024 1024))
(setopt package-upgrade-guard-max-review-files 5000)
(setopt package-upgrade-guard-max-total-diff-size (* 2 1024 1024))

;; Use Emacs' built-in archive package review support when available.
(setopt package-upgrade-guard-prefer-built-in-review t)
#+END_SRC

** Advanced Configuration

The package provides several customization options through the ~package-upgrade-guard~ group:

- ~package-upgrade-guard-enabled~ - Global enable/disable
- ~package-upgrade-guard-temp-dir~ - Custom base directory for temporary review sessions
- ~package-upgrade-guard-diff-mode~ - Show normal diffs or highlight security-sensitive patterns in complete diffs
- ~package-upgrade-guard-max-download-size~ - Maximum downloaded artifact size
- ~package-upgrade-guard-max-review-files~ - Maximum number of files compared before the review requires manual incomplete-review approval
- ~package-upgrade-guard-max-total-diff-size~ - Maximum aggregate archive diff size before the review requires manual incomplete-review approval
- ~package-upgrade-guard-prefer-built-in-review~ - Delegate archive package review to Emacs' built-in support when available
- ~package-upgrade-guard-security-diff-regexp-list~ - Regexps used by security-only diff mode
- ~package-upgrade-guard-security-active-document-regexp-list~ - Regexps for active document constructs that require manual review

Security highlighting is a review aid based on configurable heuristics, not a
proof that unmatched content is safe.  It never hides unmatched changes.
Every detected change requires manual approval.  A complete comparison with no
differences proceeds without prompting.  Incomplete reviews display the
available review buffer and require explicit manual approval.

* Requirements

- Emacs 30.2 or later
- Standard packages: ~package~ and ~diff~
- Git-based VC upgrade review requires Emacs' ~package-vc~ support and ~git~ in your PATH

* Security Considerations

Package Upgrade Guard is designed for defensive security purposes:

- Helps identify potentially malicious changes in package updates
- Provides transparency into modifications and upgrade-induced dependency changes
- Allows users to make informed decisions about package upgrades
- Shows the reviewed changes before package modifications are applied
- Requires manual approval for every reviewed change; direct new installations are outside its review scope
- Cancels installation when artifact verification or complete VC validation fails

* Limitations

- Oversized downloads cancel review; oversized review trees and diffs require explicit incomplete-review approval
- Binary files are not text-diffed (shown as modified with size changes)
- Security-only matching is heuristic and cannot prove that arbitrary Emacs Lisp is safe
- Emacs versions before 30.2 are not declared supported until tested
- Temporary files are created during analysis

* Troubleshooting

** Common Issues

*** Temporary Directory Errors
If you encounter permission errors, set a custom temporary base directory:

#+BEGIN_SRC emacs-lisp
(setopt package-upgrade-guard-temp-dir "~/tmp/package-upgrade-guard")
#+END_SRC

*** Git Errors for VC Packages
Ensure git is in your PATH and the package directory is a valid git repository.

*** Performance with Large Packages
The tool truncates large diffs automatically. For extremely large packages, consider reviewing changes externally.
